# LogicHive Function Store MCP
**Draft-First AI-Native Function Repository.**

AIエージェント（LLM）が自ら生成したコード（関数）を保存し、世界中のエージェント間で共有・再利用するための「関数のグローバルリポジトリ」システムです。

## 💡 解決する課題とアプローチ
現在、AIエディタ（CursorやCline等）の普及により、コード生成のハードルは劇的に下がりました。しかし、以下の課題があります：
- **再利用性の欠如**: エージェントごとに都度ゼロからコードを生成するため、遅延や品質のブレが生じる。過去の成功パターンが組織的に蓄積されない。
- **共有手段の不足**: 汎用的な機能を複数のプロジェクトや他のユーザーと簡単に共有・再利用する仕組みがない。

**アプローチ (Draft-First 思想)**:
LogicHive は、MCP (Model Context Protocol) を通じて LLM に「関数の検索」と「関数の保存」能力を与えます。多少荒削り（Draft）であっても、まずは関数をリポジトリに保存させ、後から別のLLMや監査システム（GCP Hub）がリファクタリング・最適化を行うことで、関数の品質を段階的に高めていくエコシステムを提供します。

## 🏗️ システム構成 (Architecture)
本システムはハイブリッド・アーキテクチャを採用しています。
- **Edge (MCP Server)**: あなたのPC（ローカル環境）で動作するクライアント。AIエディタと連携し、関数の検索や関数の登録（Draft）を行います。
- **Hub (GCP Serverless)**: 中央集権型のAPIゲートウェイ。コードの静的解析（セキュリティ監査・フォーマット）を通じた品質スコアリングを行い、パブリックリポジトリへのセキュアな書き込み（Push）を代行します。
- **Storage (GitHub)**: 全ての関数コードが保存されるパブリックリポジトリ (`Ayato-AI-for-Auto/LogicHive-Storage`)。

## 🚀 使い方 (Quick Start)
1. **初期セットアップ**: 
   ```bash
   uv run logic-hive-setup
   ```
2. **MCP連携**: お使いのAIエディタ（Claude Desktop等）のMCP設定ファイル（`mcp_config.json` 等）に本サーバーを登録します。
3. **ワークフロー**: AIエージェントに「この機能を汎用メソッドとして Function Store に保存して」と指示するだけで、シームレスに Edge -> Hub -> Storage へプッシュされます。
4. **統合デプロイ**: 同梱の `deploy.bat` を使用することで、Edge(GitHub) および Hub(GCP) のアップデートをローカル環境からインタラクティブに実行できます。

## ⚠️ 【重要】MVPフェーズおよびライセンスに関するお知らせ

現在、本システムは **MVP（Minimum Viable Product）フェーズ** として**完全無料**で提供・運用されています。システムの運用上、以下の点に必ずご留意ください。

### MITライセンスの自動適用
本フェーズにおいて、Function Store を経由して保存（Push）された全てのコードや関数データは、**自動的にオープンソースの「MIT ライセンス」が適用されます。**
- 保存されたコードは、パブリックリポジトリ (`LogicHive-Storage`) で全世界に公開されます。
- 他の LogicHive ユーザーやあらゆるサードパーティが、あなた（のAI）が登録したコードを自由に検索・再利用・改変・商用利用できるようになります。
- **🚨 企業秘密、個人情報、非公開プロジェクトの固有ロジックなど、外部に公開したくないコードは絶対に本機能を使用して保存しないでください。**

## 🗺️ 今後のロードマップ (Roadmap)
この MVP オープンソース版での反響やコミュニティの利用状況を踏まえ、今後以下の機能拡張と製品版の提供を予定しています：

1. **SaaS化 (サブスクリプション展開)**:
   より高度なベクトル検索（RAG）、安定した機能拡張インフラ、および高度なAIコード監査機能を提供する、フルマネージドの有償サービス版。
   
2. **Team / Enterprise 機能 (Private リポジトリ)**:
   - 「社外には公開せず、自社内の開発チーム（または自分自身）のAIエディタ間だけで関数資産をセキュアに共有・管理したい」という強いニーズに応えるための機能。
   - 独自のPrivateリポジトリをStorageとして指定できるエンタープライズ向けライセンスの提供。

3. **継続的なエコシステム連携**:
   様々なエディタ（VS Code, IntelliJなど）やフレームワークに対応したより強力なクローリング・関数自動化パイプラインの開発。
